MongoDB aggregation pipeline

$match — filter documents
Filters the collection. Only documents passing the condition move to the next stage. Equivalent to SQL WHERE. Always place $match early to reduce data volume.
// Only mobile products { $match: { category: "mobile" } }
name price category
Apple iPhone 14 ₹80,000 mobile
Samsung Galaxy S23 ₹75,000 mobile
// Price between 10,000 and 90,000 { $match: { price: { $gte: 10000, $lte: 90000 } } } // Operators: $gt $gte $lt $lte $ne $eq $in $nin
name price
Samsung Galaxy S23 ₹75,000
Apple iPhone 14 ₹80,000
Asus ZenBook ₹90,000
Apple Watch ₹35,000
// Mobile OR laptop over ₹70,000 { $match: { $and: [ { price: { $gt: 70000 } }, { category: { $in: ["mobile", "laptop"] } } ] } } // Logical: $and $or $nor $not
name price category
Apple iPhone 14 ₹80,000 mobile
Samsung Galaxy S23 ₹75,000 mobile
Apple MacBook Air ₹1,20,000 laptop
Asus ZenBook ₹90,000 laptop
// Products whose name starts with "Apple" { $match: { name: { $regex: /^Apple/, $options: "i" } } }
name price
Apple iPhone 14 ₹80,000
Apple MacBook Air ₹1,20,000
Apple Watch ₹35,000
// Mobile products between ₹10,000 and ₹90,000 { $match: { category: "mobile", price: { $gte: 10000, $lte: 90000 } } }
$group — aggregate by field
Groups documents by _id key and applies accumulators. The _id can be a field, expression, or null (to group everything). All accumulator operators are shown below.
{ $group: { _id: "$category", totalProducts: { $sum: 1 }, totalRevenue: { $sum: "$price" }, avgPrice: { $avg: "$price" } } }
_id totalProducts totalRevenue avgPrice
mobile 2 ₹1,55,000 ₹77,500
laptop 2 ₹2,10,000 ₹1,05,000
accessory 1 ₹2,500 ₹2,500
wearable 1 ₹35,000 ₹35,000
{ $group: { _id: "$category", cheapest: { $min: "$price" }, costliest: { $max: "$price" } } }
_id cheapest costliest
mobile ₹75,000 ₹80,000
laptop ₹90,000 ₹1,20,000
accessory ₹2,500 ₹2,500
wearable ₹35,000 ₹35,000
{ $group: { _id: "$category", allNames: { $push: "$name" }, // duplicates allowed uniqueNames: { $addToSet: "$name" } // no duplicates } }
_id allNames (array)
mobile ["Apple iPhone 14", "Samsung Galaxy S23"]
laptop ["Apple MacBook Air", "Asus ZenBook"]
accessory ["Boat Headphones"]
wearable ["Apple Watch"]
// Requires a preceding $sort to be meaningful { $sort: { price: 1 } }, { $group: { _id: "$category", cheapestProduct: { $first: "$name" }, costliestProduct: { $last: "$name" } } }
_id cheapestProduct costliestProduct
mobile Samsung Galaxy S23 Apple iPhone 14
laptop Asus ZenBook Apple MacBook Air
{ $group: { _id: "$category", priceStdDev: { $stdDevPop: "$price" }, // $stdDevSamp — sample std deviation variance: { $stdDevSamp: "$price" } } }
$stdDevPop = population std dev (all docs). $stdDevSamp = sample std dev (subset). Used in analytics to measure price spread.
_id priceStdDev
mobile ₹2,500
laptop ₹15,000
// _id: null → treat entire collection as one group { $group: { _id: null, grandTotal: { $sum: "$price" }, overallAvg: { $avg: "$price" }, productCount: { $sum: 1 } } }
_id grandTotal overallAvg productCount
null ₹4,02,500 ₹67,083 6
$sort — order documents
Sorts documents. 1 = ascending (A→Z, low→high), -1 = descending. Can sort by multiple fields — order of fields matters.

$project — shape output fields
Includes/excludes fields and creates computed fields using expressions. 1 = include, 0 = exclude. _id is included by default unless set to 0.
{ $project: { _id: 0, name: 1, price: 1 } }

Apple iPhone 14 ₹80,000
Samsung Galaxy S23 ₹75,000
Apple MacBook Air ₹1,20,000
Asus ZenBook ₹90,000
Boat Headphones ₹2,500

$limit & $skip — pagination
$limit restricts how many documents pass through. $skip discards the first N documents. Together they implement pagination.
// Page 2 of results (2 per page), sorted by price desc { $sort: { price: -1 } }, { $skip: 2 }, // skip first 2 (page 1) { $limit: 2 } // take next 2 (page 2)
name price
Apple iPhone 14 ₹80,000
Samsung Galaxy S23 ₹75,000
Page formula: skip = (pageNumber - 1) × pageSize
$unwind — flatten arrays
Deconstructs an array field into individual documents — one document per array element. Essential before grouping on array values.
// Imagine products have a tags array: // { name: "Apple iPhone 14", tags: ["5G","OLED","iOS"] } { $unwind: "$tags" } // Extended syntax (preserve nulls): { $unwind: { path: "$tags", includeArrayIndex: "tagIndex", preserveNullAndEmptyArrays: true } }
One input doc with 3 tags → 3 output docs, each with a single tag value. Use with $group after to count tag frequencies.
name tags (after unwind) tagIndex
Apple iPhone 14 5G 0
Apple iPhone 14 OLED 1
Apple iPhone 14 iOS 2
$lookup — join collections
Performs a left outer join with another collection. Equivalent to SQL LEFT JOIN. The matched documents are added as an array field.
// Join products → orders collection { $lookup: { from: "orders", localField: "_id", foreignField: "productId", as: "productOrders" } }
Each product document gets a "productOrders" array containing all matching orders. Empty array [] if no matches (left outer join).
// Pipeline join — more control, can filter joined docs { $lookup: { from: "orders", let: { prodId: "$_id" }, pipeline: [ { $match: { $expr: { $eq: ["$productId", "$$prodId"] } } }, { $sort: { orderDate: -1 } }, { $limit: 5 } ], as: "recentOrders" } }
Pipeline join lets you filter, sort and limit the joined documents before attaching them. More powerful than the basic form.
$addFields / $set — add new fields
Adds new fields (or overwrites existing ones) without touching other fields. $set is an alias for $addFields introduced in MongoDB 4.2.
{ $addFields: { gstPrice: { $multiply: ["$price", 1.18] }, priceInUSD: { $round: [{ $divide: ["$price", 83] }, 2] }, isBudget: { $lt: ["$price", 10000] } } } // Same as: { $set: { ... } }
name price gstPrice priceInUSD isBudget
Apple iPhone 14 ₹80,000 ₹94,400 $964.00 false
Boat Headphones ₹2,500 ₹2,950 $30.12 true
Apple Watch ₹35,000 ₹41,300 $421.69 false
Unlike $project, $addFields keeps ALL original fields and just adds the new ones on top.
$replaceRoot / $replaceWith — promote embedded doc
Replaces the document with a specified sub-document or expression. Useful when a nested object should become the top-level document.
// Imagine products have: { specs: { ram: "8GB", storage: "256GB" } } { $replaceRoot: { newRoot: "$specs" } } // Shorthand alias: { $replaceWith: "$specs" } // Merge with original to keep both: { $replaceRoot: { newRoot: { $mergeObjects: ["$$ROOT", "$specs"] } } }
$$ROOT refers to the current top-level document. $mergeObjects is useful to promote nested fields while keeping the parent fields too.
$bucket / $bucketAuto — histogram grouping
Groups documents into ranges (buckets). $bucket uses manual boundaries. $bucketAuto distributes evenly into N buckets automatically.
{ $bucket: { groupBy: "$price", boundaries: [0, 10000, 50000, 100000, 200000], default: "Other", output: { count: { $sum: 1 }, products: { $push: "$name" } } } }
_id (range start) count products
0 (₹0–9,999) 1 ["Boat Headphones"]
10000 (₹10k–49,999) 1 ["Apple Watch"]
50000 (₹50k–99,999) 3 ["iPhone 14","Galaxy S23","ZenBook"]
100000 (₹1L+) 1 ["Apple MacBook Air"]
{ $bucketAuto: { groupBy: "$price", buckets: 3, output: { count: { $sum: 1 }, avg: { $avg: "$price" } } } }
MongoDB automatically calculates boundaries to distribute ~equally. The _id contains {min, max} of each bucket.
_id.min _id.max count avg
₹2,500 ₹75,000 2 ₹38,750
₹75,000 ₹90,000 2 ₹82,500
₹90,000 ₹1,20,000 2 ₹1,05,000
$facet — multiple pipelines in one
Runs multiple sub-pipelines on the same input documents simultaneously. Returns a single document where each key is the result of one sub-pipeline. Perfect for search facets, dashboards.
{ $facet: { byCategory: [ { $group: { _id: "$category", count: { $sum: 1 } } } ], priceStats: [ { $group: { _id: null, min: { $min: "$price" }, max: { $max: "$price" }, avg: { $avg: "$price" } } } ], topProducts: [ { $sort: { price: -1 } }, { $limit: 3 }, { $project: { name: 1, price: 1, _id: 0 } } ] } }
Output is one document with three keys: byCategory[], priceStats[], topProducts[]. All three run on the same 6 products in parallel.
$count — count documents
Counts the number of documents at that stage and returns a single document with the count. Simpler than $group with $sum:1.
// Count products over ₹50,000 { $match: { price: { $gt: 50000 } } }, { $count: "premiumCount" }
premiumCount
4
Also useful: $sample — randomly select N documents from collection.
{ $sample: { size: 2 } } → returns 2 random products.
$out / $merge — write results
Write the pipeline result to a collection. $out replaces the target collection entirely. $merge is more flexible — can insert, merge, or replace individual documents.
// Save category summary to a new collection { $group: { _id: "$category", avg: { $avg: "$price" }, count: { $sum: 1 } } }, { $out: "category_summary" } // Drops & recreates "category_summary" collection
{ $merge: { into: "category_summary", on: "_id", whenMatched: "merge", // merge | replace | keepExisting | fail whenNotMatched: "insert" // insert | discard | fail } } // Upserts: updates existing docs, inserts new ones
$merge is safer than $out for incremental updates — it won't delete existing data that doesn't match your pipeline output.
Full pipeline — 7 stages combined
$match → $addFields → $group → $sort → $project → $skip → $limit
Business goal: "Show the top 2 categories (excluding accessories) sorted by avg GST-inclusive price, with product count."
db.products.aggregate([ // 1. Exclude accessories { $match: { category: { $ne: "accessory" } } }, // 2. Compute GST price per product { $addFields: { gstPrice: { $multiply: ["$price", 1.18] } }}, // 3. Group by category { $group: { _id: "$category", count: { $sum: 1 }, avgGST: { $avg: "$gstPrice" }, maxPrice: { $max: "$price" } }}, // 4. Sort by avgGST descending { $sort: { avgGST: -1 } }, // 5. Rename _id → category, round avgGST { $project: { _id: 0, category: "$_id", count: 1, avgGST: { $round: ["$avgGST", 0] }, maxPrice: 1 }}, // 6. Skip 0 (page 1), limit 2 { $skip: 0 }, { $limit: 2 } ])
Result
category count avgGST maxPrice
laptop 2 ₹1,23,900 ₹1,20,000
mobile 2 ₹91,450 ₹80,000
